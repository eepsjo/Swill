在较新的主板[^1]上使用新版 [Ventoy](https://github.com/ventoy/Ventoy) 可能会在安全启动下出现 `Invalid Image` 错误，该错误导致无法弹出 MOK 界面进行 enroll、即使手动启动并在 MOK 界面手动选择 Hash 注册、使用 `VTOY_SECURE_BOOT_POLICY`[^2] 都提示文件非法。经检查系新版固件强制引入 **Microsoft UEFI CA 2023** 证书并彻底废除[^1] **Microsoft UEFI CA 2011** 旧证书导致。

需要注意的是，虽然最新版 Ventoy 采用了 Rocky Linux 的双签名 `shim`，但由于其内部仍携带已被主板 `dbx`（吊销列表）强制拉黑的 2011 证书特征或旧版 Shim 漏洞代码，新版固件会在硬件层直接拦截该文件的执行。最稳妥的安全闭环方案是保持 BIOS 密码保护，在使用 Ventoy 时手动暂时关闭安全启动[^3]。

[^1]: 也有可能是老主板更新固件后重置密钥导致 2011 CA 丢失

[^2]: 见 [Ventoy GitHub Issue #3639](https://github.com/ventoy/Ventoy/issues/3639) 以及官方的 [SecureBoot.md](https://github.com/ventoy/Ventoy/blob/master/SecureBoot.md)

[^3]: 这种“物理级”主动清空并恢复默认金钥的行为，实际上帮助主板关闭了 2011 证书降级攻击的后门。配合 BIOS 密码，即使临时关闭安全启动，整机的硬件信任链依然处于极高规格的闭环保护下
