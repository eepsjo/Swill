# fix aria2 80092013

使用[原版 aria2](https://aria2.github.io/) 下载文件时可能出现`SSL/TLS 握手错误：吊销服务器脱机，吊销功能无法检查吊销`问题，经检查[^1]系 `WinSSL` 的问题，使用[这个仓库](https://github.com/dmesg00/aria2-static-builds)提供的使用 `OpenSSL` 的 aria2 即可在配置文件不添加 `check-certificate=false`[^2] 的情况下解决吊销问题。

需要注意的是由于使用 `OpenSSL` 证书并不存在于 Windows，需要手动指定本地证书，而仓库也提供了这样的证书，可以在启动时通过参数 `--ca-certificate=ca-certificates.crt` 调用（示例中证书名为 `ca-certificates.crt` 且和 aria2 在同一目录），也可以更改配置文件使用。

[^1]: 检查包括不限于：

    - GFW 是否屏蔽/污染/干扰 Let's Encrypt？

    - 曾经执行过的系统优化是否影响 Windows 的检查证书吊销行为？

    - 防火墙规则/hosts

    - aria2 流量是否被 TUN 正确代理

[^2]: 绕过证书检查。可能引起 MitM 问题。即使没有 MitM 风险，下载完成后校验哈希值也是必需的。
