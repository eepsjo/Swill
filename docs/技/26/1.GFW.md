# GFW

GFW 主要通过以下几种方式来识别和阻断用户访问被封锁的网站：

1. **SNI 阻断**：通过监测 TLS 握手中明文的 SNI 字段，识别用户访问的域名，并进行阻断；
2. **IP 封锁**：直接封锁网站服务器的 IP 地址，阻断所有对该 IP 的访问；
3. **DNS 劫持与污染**：通过篡改 DNS 响应，阻止用户解析被封锁网站的域名；
4. **深度包检测（DPI）**：分析网络流量中的特征，识别并阻断特定的协议或应用。

现有的“魔法”协议都建立在以下“美好幻想”（但一定程度上是事实）上：

1. GFW 将长期在大部分地区保持黑名单的状态；
2. GFW 将长期只封锁明确用于代理的 IP 地址和域名；
3. GFW 将长期只封锁违规的网站的 IP 地址和域名；
4. GFW 将长期只封锁明确用于“魔法”的协议特征；
5. GFW 将长期无法准确干预 IPv6 流量和 udp 流量；
6. GFW 将长期无法跟上最先进的“魔法”技术。

这就导致：

1. VPS 的 IP 只要不在黑名单上，就能长期使用；
2. 新域名往往不会立即被封锁，可以用来搭建代理服务；
3. 使用 CDN（如 Cloudflare）等服务，可以利用其庞大的 IP 池和域名掩护，绕过 GFW 的封锁。

于是有了：

1. 基于域名掩护的“魔法”方式（过 CDN，保护服务端 IP）。此方法只需要确保 CDN 的 IP 地址未被封锁即可，域名可以随意更换（或使用 ECH）；
2. 基于模拟伪装（鹦鹉）的“魔法”方式（如 Reality）。此方法甚至不需要域名，但由于 Reality 模拟的是完整的 TLS 握手，因此不能套 CDN，也就必须保证所使用的服务器 IP 地址未被封锁。
3. 基于 udp 的协议（如 Hysteria）。此方法利用 GFW 对 udp 流量干预较少的漏洞。

GFW 只需要全境 SNI 和 IP 白名单，上述所有“魔法”协议都将失效。目前有 SNI 白名单的趋势，且上述三种方式都不再有很大成效。
