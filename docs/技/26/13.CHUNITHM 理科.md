利用上面的脚本

1. 实现在游戏启动前弹窗询问[^1]并编辑 `segatools.ini` 实现换服；

[^1]: 通过任务栏的 Playnite 小菜单启动时会直接绕过弹窗，也就是不会修改 `segatools.ini`，保留上一次弹窗的结果。

2. 如果启动时系统时间在 02:30 至 06:00，强行设置时间为 06:15 来绕过 02:45 至 06:00 的“メンテナンスのため、プレイ受付は終了しています”[^2]。

[^2]: 写死在中二节奏代码中的“服务器维护”时间，期间无法游玩。

**对于 1：**服务器的切换基于对 `segatools.ini` 的编辑；不同服的 `keychip` 号也不同。事先在 `segatools.ini` 写好两个服务器的配置（先注释掉其中之一），例如：

> **注意：**编辑 `segatools.ini` 时应当只修改相关的键值，或参考[这篇教程](https://unreal-network.tech/docs/arcade/segatools/)操作。

``` ini
[dns]
; AquaNET
default=aime.my-aqua.net
; RinNET
;default=aqua.naominet.live

[keychip]
; AquaNET
id=A61E-00000000000
; RinNET
;id=A39E-00000000000
```

随后执行启动前脚本时会根据弹窗选择结果添加/删除注释，实现换服。

**对于 2：**事先在 `bin` 文件夹下创建 `TimeTravel.ps1`：

``` powershell
param(
    [string]$Action
)
$OffsetFile = Join-Path $PSScriptRoot "time_offset.txt"
$maintStart = New-TimeSpan -Hours 2 -Minutes 30
$maintEnd = New-TimeSpan -Hours 6 -Minutes 0
if ($Action -eq "Start") {
    $now = Get-Date
    $startTime = $now.TimeOfDay
    if ($startTime -ge $maintStart -and $startTime -lt $maintEnd) {
        if (-not ([Security.Principal.WindowsPrincipal][Security.Principal.WindowsIdentity]::GetCurrent()).IsInRole([Security.Principal.WindowsBuiltInRole]::Administrator)) {
            Start-Process powershell -ArgumentList "-NoProfile -ExecutionPolicy Bypass -File `"$PSCommandPath`" -Action Start" -Verb RunAs -WindowStyle Hidden
            exit
        }
        $targetTime = Get-Date -Hour 6 -Minute 15 -Second 0
        $TimeOffsetTicks = ($targetTime - $now).Ticks
        $TimeOffsetTicks | Out-File $OffsetFile -Force
        Set-Date $targetTime | Out-Null
    }
}
if ($Action -eq "Restore") {
    if (Test-Path $OffsetFile) {
        if (-not ([Security.Principal.WindowsPrincipal][Security.Principal.WindowsIdentity]::GetCurrent()).IsInRole([Security.Principal.WindowsBuiltInRole]::Administrator)) {
            Start-Process powershell -ArgumentList "-NoProfile -ExecutionPolicy Bypass -File `"$PSCommandPath`" -Action Restore" -Verb RunAs -WindowStyle Hidden
            exit
        }
        $TimeOffsetTicks = [long](Get-Content $OffsetFile -Raw)
        $realNow = (Get-Date).AddTicks(-$TimeOffsetTicks)
        Set-Date $realNow | Out-Null
        Remove-Item $OffsetFile -Force
    }
}
```

该脚本会在“启动前脚本”和“退出后脚本”中分别执行，自我判断是否需要执行并提权 UAC。

### 启动后脚本

if (Test-Path "..\Brokenithm\start.vbs")

利用上面的脚本实现隐式启动 Brokenithm。

对于 Brokenithm，在 `segatools.ini` 中可能需要额外配置：

``` ini
[chuniio]
; If you wish to sideload a different chuniio, specify the DLL path here
;path=chuniio.dll

; YubiDeck use this
;path=yubideck.dll

; TASOLLER use this
;path=tasoller.dll

; TASOLLER PLUS use this
;path=tasoller_plus.dll

; Brokenithm use this
path=brokenithm.dll

; chuniio-mux.dll is a aggregate version of io
; this will mount all the above io at the same time
; and no need to adjust any configuration
; you can use Yubideck/TASOLLER/TASOLLER PLUS/Brokenithm and keyboard at the same time
; if you have any problem after mount this dll, please try to mount the dll you need only
; and please do not rename the dll, otherwise it will not be loaded
;path=chuniio-mux.dll
```

可以防止启动中二后键盘误操作输入（只使用 Brokenithm 操作中二）[^3]。

[^3]: Brokenithm 的主流用法是直接放到 `bin` 然后修改 `start.bat` 启动。

    由于部分版本 Brokenithm 自带 `aimeio.dll`、`chuniio.dll`和`brokenithm.dll`，直接放入会覆盖这些 segatools 可能自带的 dll，也就是配置安装理科时放入 `bin` 的文件。
    
    实测独立文件夹运行、不修改 `start.bat` 也是可行的：只要在 `segatools.ini` 启用了 `brokenithm.dll` 或 `chuniio-mux.dll`。

此外，现版本的 `aimeio` 总会尝试刷入 20 个 `0` 的 aime 号。最佳实践是禁用一切 `aimeio` dll[^4]。

[^4]: [来源](https://bbs.xqemu.cn/thread-9427-1-1.html)。一旦禁用，无法使用 Brokenithm 的刷卡功能，只能通过键盘回车键刷卡。

    实际使用中发现：1、如果独立运行 Brokenithm，也就是不把它放进 `bin`，无论是否启用 `aimeio` Brokenithm 都无法刷卡（说明 segatools 现版本的 `aimeio.dll` 与 Brokenithm 不兼容）；2、放入 `bin` 的运行（覆盖 `aimeio.dll`）可以使用刷卡，但也无法解决疯狂刷 `0` 的问题。

``` ini
[aimeio]
; Specify the DLL path if you wish to sideload a different aimeio.
; When using any aimeio.dll, ensure `enable=1` is set in the [aime] section.

;path=aimeio.dll

; YubiDeck (DASI)
;path=aimeio_yubideck.dll

; HINATA Aime Reader (Aimeio Mode)
; if you need get latest hinata info or you need to update hinata aimeio dll, please go to hinata.neri.moe
;path=aimeio_hinata.dll
```

### 退出后脚本

``` powershell
$TargetProcesses = @("Brokenithm-Evolved-iOS")
Stop-Process -Name $TargetProcesses -Force -ErrorAction SilentlyContinue

$BinDir = "bin"
# 同上文
$TimeScript = "$BinDir\TimeTravel.ps1"

if (Test-Path $TimeScript) {
    Start-Process powershell -ArgumentList "-NoProfile -ExecutionPolicy Bypass -File `"$TimeScript`" -Action Restore" -WindowStyle Hidden
}
```

利用上面的脚本实现退出 Brokenithm 服务端，并按需执行 `TimeTravel.ps1` 以切换回正确系统时间。
