## 模块“编辑参数”

### mode & url-test & fallback

`mode` 可选值有 `url-test` 和 `fallback`，需要手动填写完整键值，代表模块将以那种模式代理 APNs 流量。缺省值 `fallback`。

`url-test` 中可填入一个或多个代理节点或分组的名称，每个节点或分组之间用英文逗号分隔。这些节点将进行 URL 测试，测试结果最好的节点将被用来代理 APNs。不填则使用缺省值 `PROXY`。建议填写一个或多个稳定、低延迟的节点。

`fallback` 可填入一个或多个代理节点或分组的名称，每个节点或分组之间用英文逗号分隔。这些节点将在 `url-test` 中的节点都不可用时被使用。不填则使用缺省值 `DIRECT`，也建议使用缺省值。

> 手动指定固定的节点，目的是防止切换配置时/代理组切换节点时打断 APNs 长连接导致信息接收延迟。<br>直连作为回落兜底防止节点全部超时导致 APNs 完全失效（至少直连时不需要绕过 APNs 屏蔽的应用还能收到消息）。

### interval & tolerance & timeout & url

上述模式参数，一般不需要手动设置。具体用法参照[箭箭火使用文档](https://lowertop.github.io/Shadowrocket/)。

### init-?01st.push.apple.com

是否代理 `DOMAIN-WILDCARD,init-?01st.push.apple.com`。默认关闭（开启表示代理，关闭以切换到直连。下同）。

> 在兜底代理配置的 [D 规则集](https://raw.githubusercontent.com/Loyalsoldier/clash-rules/refs/heads/release/direct.txt)中存在两条直连规则

   ``` yaml
   payload:
     - 'init-p01st.push.apple.com'
     - 'init-s01st.push.apple.com'
   ```

> 与下一条参数冲突，故创建此选项让用户自行定夺。

### +.push.apple.com

是否代理 `DOMAIN-SUFFIX,push.apple.com`。默认开启。

### IPs

是否代理 APNs 的 IP 段。默认开启。

参考了 [Apple 的官方文档](https://support.apple.com/102266)，这里说明了 APNs 的 IP 段。

---

模块配置完成后，开启关闭飞行模式一次（这是网传的方法，目的是彻底关闭再开启一次箭箭火隧道）即可复活 APNs。

## 原理

基于笔者对 APNs 屏蔽的技术原理的*猜测*：*笔者认为，APNs 直接通过 APNs 连接检测用户的 IP 来确定用户是否在墙内，以此屏蔽指定应用的消息推送。*因此让 APNs 服务器认为我们在境外就能复活 APNs。
