# 浅谈系统更新

> 今天在 bilibili 刷到一个视频，UP 主通过绘制散点图并结合微软历史上的 Windows 系统更新规律预测下一代 Windows 或已初具雏形。针对评论区以及很多人禁用系统更新的行为我有点想说的。

## 一、Operation Systems

### 1. 窗边瞳子和她的姐妹们

我从 Windows 95 到现在最新一代 的 11 均有过使用经历（XP 和 8.1 是我最喜欢的两个大版本，现在正在使用 10）。几年前，我曾参与过预览体验计划的 Dev 渠道，并在第一时间安装尝试了 22000；也曾尝试过 Windows 10X，不过这个系统只能通过 MS Store 的专用虚拟机软件下载尝试。

在我使用 Windows 10 之前的很长一段时间，我对系统更新这个概念是无感的，只知道 Windows 每隔几年会迭代出新版本，却不知道每个大版本内是有很多小版本或小更新的。就我所知，除去换版本号的大更新，在 10 中 Windows Update 主要分为质量更新、定义更新和其他类型的更新（驱动程序更新和其他一些有的没的，比如“恶意软件删除工具”、“语言包”、“.NET Framework”。驱动程序更新往往是厂家而非微软推送，比如主板 BIOS 更新，这里的驱动不包括用户自己安装的驱动，因为自己安装的算是软件而不是系统组件了）。

换版本号的更新感觉就像下了个新版本的镜像文件覆盖安装 Windows，更新重启很多次不说，界面和质量更新也大有不同，有时会因为短暂的没启用部分驱动而让显示器显示的怪怪的（比如夜览功能不生效且屏幕亮度最高）。更新完成 “C:\” 会多出一个 Windows.old 文件夹，里面存放了更新前的数据，因此可以用设置中的“回退 Windows 版本”滚回去。

质量更新也是要重启电脑的，但是这种重启和换版本号的更新不同，它一般只重启两次，一次是更新开始，另一次在 33% 左右，且往往很快，显示的界面是六个点转圈圈的。这些更新往往没有大问题，而且很可能是修复安全问题的（安全问题是这篇文章的重点）。这些更新回滚也快，卸载对应 KB 号的更新包就行。

> 提一嘴防止有人不懂……重启在 Windows 中和关机后开机不太一样。如果电脑开了“快速启动”的话。

其他种类的更新基本不需要重启就能完成。

### 2. 你是湊企鹅你是湊企鹅

> 我用过很多发行版，但是真正作为日常使用的只有 Ubuntu。我们这里不去争论 GNU/Linux 等称谓的意义和叫法，我就仅对 Ubuntu 这一发行版的更新讨论。

用过 Linux 的都知道很多发行版有自己的包管理器，包管理器一般可以 CLI 输入命令快速管理电脑内的软件，比如 apt update；而很多系统组件也通过包管理器更新，所以 update 完后 upgrade 一下就更新了很多东西。

Ubuntu 的发行版分 LTS 和普通的，LTS 能用一个叫做 Ubuntu Pro 的功能获取额外的安全更新，此外 Ubuntu 自带一个可以定期检测软件更新的 GUI 程序，这个程序也可以更新 Linux 内核。事实上内核版本的更新和发行版的更新是基本相互独立的，毕竟发行版的本质是在内核上预制安装软件和桌面环境（比如 Wayland、Gnome、LibreOffice）来达到开箱即用的效果；发行版可能是公司维护的，但内核和其他系统组件是广大开源社区贡献者维护的。

### 3. Shot on iPhone

MacOS、iOS 和 iPadOS。由 Apple 完美管理，因为系统和生态闭源而闭塞，不需要保持系统最新而能维持相对安全的状态。系统更新几乎纯粹是新功能更新，老设备出现非常严重的安全漏洞时 Apple 有可能发送针对低版本的补丁包。

## 二、更新的价值

首先是兼容性，如果你是臭打游戏的你就知道有些软件连 Windows 7 都不兼容了；当然很多生产力工具也是（这也是为啥使用 Linux 的人还是很少，大多数人由于工作需要被迫吃微软的【已过滤】和受微软视奸）。

**其次是安全**，这是最容易被普通用户忽视，却又最触及核心利益的一点。很多人觉得“我又不存机密文件，黑客黑我图什么”；或迷信杀软；或压根不在意网络安全。私认为，系统更新最重要的是修补漏洞。

> 好吧，我确实有点自我矛盾，我一方面不想受大公司或其他任何组织个人的影响，另一方面又不得不依赖安全更新。因为互联网早就脱离其本质和精神往中心化发展了……

在 Windows ，很多质量更新其实是为了封堵那些例如可能导致远程代码执行（RCE）的后门。你可能只是开着电脑挂着下载，甚至没点开任何可疑链接，黑客就能利用未修复的漏洞（如 SMB 1.0）或某个系统服务的缺陷（几年前的打印机服务）潜入你的系统。

这种安全风险在 Linux 环境下同样存在，即便 Linux 的权限管理极其严格，但内核层面的漏洞一旦被利用，提权轻而易举。这就是为什么 Ubuntu Pro 会强调“额外安全更新”的原因——它是在给 LTS 用户加防弹衣。

哪怕是 iOS，也曾出现过形如通过一条特定的 iMessage 就能让手机彻底沦陷的“零点击”漏洞。Apple 紧急推送的补丁往往就是为了跑赢那些正在寻找猎物的漏洞利用程序。此外你可能刷到过利用 iOS 系统漏洞实现“设计者设计之外”的功能的帖子或视频，有的需要描述文件和根证书实现。切记：“没有完美的系统；最大的漏洞是人类本身”；再好的安全措施或沙盒机制也防不了用户的灵机一动小巧思。

## 三、我也要更新吗？

很多人因为讨厌 Windows 强制重启或更新后更难用而禁用更新，说实话像是为了省点事儿而把家里的防盗门锁给拆了。现代系统的更新机制虽然有时显得激进（尤其是微软那“不更新就不让关机”的脾气），但其背后逻辑是建立在对全球网络安全态势的实时防御之上的。

作为一个折腾过 Dev 渠道、也体验过 10X 的窗边瞳子钉子户，我的建议其实很明确且不难实现：

- 对于 Windows 用户：

  - 把自动更新开回来，但是可以把自动安装关了（相当于让电脑定期自动检查并下载更新）。善用“暂停更新”功能来规避工作繁忙期，而不是直接通过注册表或来历不明的第三方工具彻底“阉割”掉更新服务。

  - 质量更新必须装。它们是你的第一道防线且通常不导致系统崩溃或 UI 巨变。如果更新后有神秘 bug 自行上网查找相应资料，或在更新前查看对应 KB 号的更新包有哪些改动。
  
  - 那种“跨版本”的大更新，完全可以等它推送半年、社区口碑稳定后再上车（或者在不影响兼容性的情况下不更新）。

  > 和我一样是窗边瞳子钉子户的可以考虑转 LTSC 或注册 ESU。

- 对于 Linux 用户：

  - 感觉不用我多说，XD

- 对于 Apple 用户：

  - 可以对大版本迭代保持观望，但对于那种只有几十或几百 MB 的“安全性响应”补丁，务必第一时间安装。

## 结

系统更新不仅仅是功能的堆砌，更是一场开发者与潜在威胁之间的博弈。我们追求系统的稳定，不是追求系统的“静止”。在这个数字安全十分重要的时代，保持系统处于最新的受支持状态，是我们作为数字公民最低成本的“保险”。
