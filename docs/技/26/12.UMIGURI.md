## Brokenithm

### iOS

### PC

需要先运行一个服务端，再连接开启了客户端的 iOS 设备。

[这个仓库](https://github.com/pinapelz/brokenithm-evolved-ios-umi)默认预设好了适配 UMIGURI 的键位和 LED 灯效果（需在 UMIGURI 中配置 Port）。

如果连接无反应，可能是电脑没有 iOS 相关驱动。[这个仓库](https://github.com/NelloKudo/Apple-Mobile-Drivers-Installer#download--installation)提供了一种免 iTunes 安装驱动的方法。

## Playnite

利用 Playnite 的 Powershell 脚本功能实现启动 UMIGURI 时协同启动 Brokenithm 服务端，并在 UMIGURI 结束时关闭。

btw，Brokenithm 服务端启动时会有一个 cmd 黑框，连接客户端后会弹出第二个，下面的方法利用 vbs 和重构 exe（需要 .NET）解决这两个黑框。

方法示例的文件结构：

```
└ UMIGURI
  ├ Broken # Brokenithm 服务端工作目录
  | ├ Brokenithm-Evolved-iOS-Umi.exe # 服务端
  | ├ start.vbs # 用于隐式启动服务端
  | ⋮
  | └ umi-evolved
  |   ├ real-brokenithm-evolved-umi.exe # 原版 exe
  |   ├ brokenithm-evolved-umi.exe # 使用 .NET 重构的影子 exe
  |   ⋮ # 按道理啥也没有，但是使用重构的 exe 后会多一个和上一级目录一样的配置文件。不影响使用
  └ UMIGURI_NEXT # UMIGURI 工作目录
    ├ UMIGURI.exe
    ⋮
```

注意到 `...\UMIGURI\Broken\umi-evolved\` 下我把原版（原名 brokenithm-evolved-umi.exe）重命名了，用以区别。

Playnite 启动前脚本：

``` powershell
if (Test-Path "Broken\start.vbs") {
    Start-Process "wscript.exe" -ArgumentList "`"start.vbs`"" -WorkingDirectory "Broken" -WindowStyle Hidden
}
```

退出后脚本：

``` powershell
$TargetProcesses = @("Brokenithm-Evolved-iOS-Umi", "brokenithm-evolved-umi", "real-brokenithm-evolved-umi")
Stop-Process -Name $TargetProcesses -Force -ErrorAction SilentlyContinue
```

start.vbs：

```
Set WshShell = CreateObject("WScript.Shell")
WshShell.Run "Brokenithm-Evolved-iOS-Umi.exe", 0, False
```

用于重构 exe 的 Powershell：

``` powershell
$TargetDir = "...\UMIGURI\Broken\umi-evolved" # 影子 exe 的最终输出目录，自己改
$RealExeName = "real-brokenithm-evolved-umi.exe" # 真正被隐藏的文件名
$ProxyName = "brokenithm-evolved-umi.exe"
$outputExe = Join-Path $TargetDir $ProxyName
$source = @"
using System;
using System.Diagnostics;
class Program {
    static void Main() {
        ProcessStartInfo startInfo = new ProcessStartInfo();
        startInfo.FileName = "$RealExeName";
        startInfo.WorkingDirectory = AppDomain.CurrentDomain.BaseDirectory;
        startInfo.WindowStyle = ProcessWindowStyle.Hidden;
        startInfo.CreateNoWindow = true;
        try {
            Process.Start(startInfo);
        } catch {}
    }
}
"@
$csc = (Get-ChildItem "$env:windir\Microsoft.NET\Framework64\v4.0.*" -Filter "csc.exe" -Recurse | Select-Object -First 1).FullName
if (-not $csc) {
    Write-Error "未找到 csc.exe 编译器，请检查是否安装了 .NET Framework 4.0 或更高版本。"
    exit
}
$tempCsFile = Join-Path $env:TEMP "broken_proxy.cs"
Set-Content -Path $tempCsFile -Value $source -Encoding utf8
if (-not (Test-Path $TargetDir)) {
    New-Item -ItemType Directory -Path $TargetDir -Force | Out-Null
}
& $csc /target:winexe /out:$outputExe $tempCsFile
Remove-Item $tempCsFile -ErrorAction SilentlyContinue
Write-Host "========================================" -ForegroundColor Green
Write-Host "编译完成！" -ForegroundColor Green
Write-Host "影子程序已生成至: $outputExe" -ForegroundColor Cyan
Write-Host "提示：请确保 $RealExeName 放在与该影子程序同级的目录下。" -ForegroundColor Yellow
Write-Host "========================================" -ForegroundColor Green
```
